## 2. App access (Sign-in details)
Die App hat geschützte Bereiche (Login/Konto). Wähle:
- **„All or some functionality is restricted"**
- Eine Anweisung hinzufügen → Demo-Zugang fürs Review-Team:
  - Name: `Login / Konto`
  - Benötigt Login: **Ja** → lege einen **Testaccount** an (E-Mail + Passwort) und trage ihn ein.
  - Hinweistext: „Most features are usable as guest (no login). To test account
    sync/premium, use the provided test login."
- *(Wenn ihr wollt, dass alles ohne Login testbar ist: „All functionality is
  available without restrictions" — aber dann sieht der Prüfer keine Konto-Features.)*

## 6. Data safety
Sag an: welche Daten werden erfasst?
- **Erhoben/geteilt:**
  - **E-Mail-Adresse** — gesammelt, *nicht* geteilt, verknüpft mit Nutzer, Zweck: Kontoverwaltung. Optional? Ja (Gast geht ohne).
  - **Name** — gesammelt, nicht geteilt, verknüpft, Zweck: App-Funktion. Optional.
  - **App-Aktivität** (Planung/Präferenzen) — gesammelt, nicht geteilt, Zweck: App-Funktion.
- **Verschlüsselung bei Übertragung:** **Ja** (HTTPS).
- **Nutzer kann Löschung beantragen:** **Ja** → URL/Hinweis: In-App „Konto löschen"
  + `info@menja-fresh.com`.
- Kein Tracking für Werbung; keine Weitergabe an Dritte.
